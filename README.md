# 🏊 Tri Coach

来年5月のトライアスロン（オリンピックディスタンス）参加に向けた、**減量・トレーニング管理アプリ**。
iPhone のホーム画面に追加して、ネイティブアプリのように使える単一ファイルの PWA です。

## できること

| 機能 | 内容 |
|---|---|
| 📊 **ホーム** | 本日のカロリー収支（摂取 − 消費）、体重推移グラフ、レースまでのカウントダウン、今週のトレーニング量 |
| ⚖️ **体重** | 体重の記録・推移グラフ（目標ライン付き）・増減の履歴 |
| 🍽️ **食事** | 文字入力（食品DB検索で自動カロリー）または写真入力でカロリー記録 |
| 🏋️ **練習** | スイム / ラン / バイク / 筋トレを記録。時間・距離・強度から**消費カロリーを自動計算**（MET法） |
| ⚙️ **設定** | プロフィール、目標体重、レース日、減量ペース、BMR/TDEE 自動算出、データのバックアップ |

## カロリー計算の仕組み

- **基礎代謝(BMR)**: Mifflin-St Jeor 式（身長・体重・年齢・性別）
- **維持カロリー(TDEE)**: BMR × 活動係数
- **消費カロリー**: `MET × 体重(kg) × 時間(h)`。ラン/バイクは距離と時間からペースを判定して MET を自動選択
- **収支**: 摂取 −（維持 + 運動消費）。設定した1日の目標収支（例 −500kcal）との達成度を表示

## 食事の写真カロリー計算

写真からカロリーを記録する方法は2通りあります。

### 方法A: Claude アプリで計算 →「🤖 貼り付け」で一括記録（おすすめ）

Claude アプリと Tri Coach は別アプリのため自動連携はできませんが、**コピペ2タップ**で橋渡しできます。

1. iPhone の **Claude アプリ**で料理を撮影し、こう頼む:
   > 「Tri Coach形式で、品名とkcalだけを1行ずつ出して」
2. こんな返答が返ります → コピー:
   ```
   親子丼 680
   サラダ 50
   味噌汁 40
   ```
3. Tri Coach の「食事」→「🤖 貼り付け」にペースト →「まとめて記録」

- 行頭に `朝/昼/夕/間` を付ければ区分も指定できます（例 `朝: オートミール 150`）。
- 番号・箇条書き記号・`kcal` の表記が混ざっていても自動で解釈します。

### 方法B: アプリ内の AI 推定（任意・API キー要）

「設定」に **Claude API キー** を入力すると、Tri Coach 内で写真から総カロリーを直接推定します（`api.anthropic.com` に送信）。
キーは端末内にのみ保存される任意機能です。キー無しでも、写真を添付して手入力で記録できます。

## iPhone での使い方

1. GitHub Pages などで公開した URL を **Safari** で開く
2. 共有ボタン → **「ホーム画面に追加」**
3. アイコンから起動するとフルスクリーンのアプリとして使えます（オフライン対応）

### GitHub Pages で公開する

リポジトリの **Settings → Pages** で、Branch を `main`（または本ブランチ）/ root に設定すれば
`https://<ユーザー名>.github.io/iphone/` で公開されます。

## データについて

- データは**この端末のブラウザ内**（localStorage / IndexedDB）に保存されます。サーバー送信はありません（AI推定を使う場合を除く）。
- 機種変更やバックアップには、設定画面の **エクスポート / インポート（JSON）** を使ってください。

## ファイル構成

```
index.html            アプリ本体（HTML/CSS/JS すべて内包）
manifest.webmanifest  PWA マニフェスト
sw.js                 Service Worker（オフラインキャッシュ）
icon.svg              アプリアイコン
```
